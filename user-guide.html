2 Log Viewer

Viewing Log Files

The Log Viewer is your primary workspace for reading, searching, and filtering a single log file at a time.

Selecting a File

Use the dropdown at the top of the viewer to switch between loaded files. The viewer instantly renders the selected file with colour-coded log levels.

Log Level Colours

LevelColourMeaning
ErrorRedErrors, exceptions, failures, and critical issues
WarningAmberWarnings that may need attention
InfoBlueInformational messages and routine operations
DebugGreyVerbose debug and trace output

Toolbar Features

üîç

Search

Type any text or regex pattern (/pattern/) to find matches. Use the arrow buttons or Enter / Shift+Enter to jump between matches.

üì°

Live Tail

Monitor a log file in real time. New lines appear at the bottom with a subtle highlight animation ‚Äî just like tailing a file in a terminal, but visual. Desktop

üîó

Stitch Files

Merge multiple log files into a single, chronologically sorted timeline. Each source file gets its own colour label so you can always see where a line came from.

üé®

Highlight Rules

Create custom colour rules (like BareTail) to visually flag specific keywords or phrases. Full details in the Highlight Rules section.

Filtering Logs

Below the search bar, you'll find filter chips to narrow down what's shown:

  • Log Level: All, Error, Warning, Info, Debug
  • Engine Type: Business, Comms, Integration
  • Time Range: Set "From" and "To" date/time to view a specific window

Footer Controls

At the bottom of the viewer you'll find the date sort toggle (Original Order, Oldest First, Newest First), a Go to Line input, and a Fullscreen button.

3 Compare Files

Side-by-Side Comparison

View multiple log files at once. Scroll them in sync, highlight differences, search across all files simultaneously, and use time-based synchronisation to correlate events.

Getting Files into Compare

Drag files onto the Compare page, or use the Add Files button. Files loaded from the Home page are also available here. Each file gets its own scrollable panel.

Compare Toolbar

1

Sync Scroll

Lock all panels so they scroll together. Invaluable when you want to read through multiple files at the same pace.

2

Time Sync

Click any line with a timestamp and every other panel jumps to the same moment in time. Use the Time Window slider (1‚Äì60 seconds) to control how close a match needs to be. Press T to collapse the panel when you need more space.

3

Highlight Differences

Compares every line across files and colours them: red for lines unique to one file, blue for lines that appear in multiple files. Best used when comparing two versions of the same file (e.g., two config files, or the same log from before and after a change).

4

Search Across All Logs

Type in the search bar to highlight matching text across every panel in real time. Toggle the filter button (funnel icon) to switch between highlight mode (show all lines, highlight matches) and filter mode (show only matching lines).

5

Layout & Fullscreen

Switch between side-by-side, tiled (2×2 grid), and horizontal layouts. Hit Fullscreen to use the entire screen for comparison.

Panel Controls

Each file panel has three buttons in its header:

ButtonWhat It Does
MinimizeCollapses the panel into a floating dock at the bottom. The remaining panels grow to fill the space. Click the chip in the dock to restore it, or use Restore All.
MaximizeExpands this panel to fill the entire compare area. Press Esc to return to normal view.
RemoveRemoves the file from the comparison entirely.

5 Highlight Rules

Custom Text Highlighting

Create BareTail-style colour rules that flag specific keywords or phrases in your logs ‚Äî in the Viewer, in Compare, and during Live Tail.

Creating a Rule

1

Open the Modal

Click Highlight Rules on the Log Viewer or Compare toolbar.

2

Define the Pattern

Type the text you want to highlight (e.g., timeout or SQL Exception). Toggle case sensitivity if needed.

3

Pick Colours

Choose a text colour and background colour. The live preview updates as you adjust. For example: black text on a yellow background makes warnings impossible to miss.

4

Choose Target Files

Select All Files to apply everywhere, or pick specific files. Useful when a keyword means different things in different engines.

5

Add & Done

Click Add Rule to save. Add as many rules as you need. Click Done to close and apply them to your logs.

Rules are saved in your browser, so they persist between sessions. You can enable/disable, edit, or delete individual rules at any time.

6 FAQs / Knowledge Base

Built-In Troubleshooting Guide

Even if an issue isn't in your current log files, the FAQs page gives you a searchable knowledge base of known Traka problems and their solutions.

How to Use

Navigate to FAQs in the sidebar. Every known issue pattern is listed as an expandable card showing:

  • Severity & Category ‚Äî at a glance, via colour-coded badges
  • "Detected in Logs" badge ‚Äî if this issue was actually found in your loaded files
  • Why it happens ‚Äî plain-English explanation
  • Prerequisites ‚Äî what you need before attempting the fix
  • Resolution Steps ‚Äî numbered, actionable steps (with commands where relevant)
  • Related Issues ‚Äî quick links to related FAQ entries

Use the search bar to find entries by keyword, or the category buttons (Configuration, Security, Database, etc.) to filter by type.

‚ÑπÔ∏è

The FAQs are a living reference. They include solutions for issues you might encounter in the future ‚Äî not just what's in today's log files. Use it proactively when troubleshooting.

9 Tips & Best Practices

Getting the Most Out of the Analyzer

A few recommendations from the team that built it.

üí°

Start with Auto-Discovery

If you're using the Desktop edition on a Traka server, let the auto-scanner find your logs. It picks the newest file from each engine automatically ‚Äî no hunting through folders.

üí°

Use Time Sync for Correlation

When an event in one engine triggers a problem in another, Time Sync is your best friend. Click a timestamp and watch all panels jump to the same moment.

üí°

Highlight Rules for Recurring Investigations

If you regularly look for the same keywords (e.g., a specific door controller ID or user name), set up highlight rules. They persist between sessions, so they're always ready.

üí°

Stitch for Chronological Analysis

When you need to understand the sequence of events across multiple engines, stitch the files together into a single timeline. Colour labels show which engine each line came from.

üí°

Check the FAQs First

Before diving deep into a log file, check the FAQs page. The issue you're seeing might already have a known solution with step-by-step instructions.

üí°

Collapse the Sidebar for More Space

When you're deep in log analysis, collapse the sidebar to give your log panels maximum width. Everything on the page scales smoothly to fill the space.
